## 1. 概述

### 1.1 文档目的
本文档详细定义了G-Zang系统中所有数据字段的含义、类型、约束条件和业务规则，为开发、测试、运维和业务人员提供统一的数据标准。

### 1.2 数据字典结构
- **字段名**：数据库字段名称
- **类型**：数据类型及长度
- **约束**：字段约束条件
- **说明**：字段业务含义
- **示例**：字段值的示例
- **关联**：与其他字段或表的关联关系

## 12. 数据约束规则

### 12.1 必填字段约束
- 所有主键字段（id）必须有值
- create_time, update_time 字段自动填充
- 标记为NOT NULL的字段必须提供值

### 12.2 数据格式约束
- 手机号：11位数字，以1开头
- 邮箱：符合标准邮箱格式
- 金额：正数，保留2位小数
- 百分比：0.00-100.00，保留2位小数

### 12.3 业务逻辑约束
- 用户余额不能为负数（可配置）
- 交易时间不能超过当前时间30天
- 预算使用率不能超过100%（可配置）
- 企业用户必须属于某个公司
