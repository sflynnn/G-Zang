# G-Zang (归藏) 部署运维手册

## 1. 环境规划

### 1.1 环境类型

#### 1.1.1 开发环境 (Development)
- **用途**：开发调试、单元测试
- **配置**：最低配置要求，快速启动
- **数据**：测试数据，允许数据丢失
- **访问**：仅开发团队内部访问

#### 1.1.2 测试环境 (Testing)
- **用途**：集成测试、系统测试、验收测试
- **配置**：与生产环境一致
- **数据**：生产数据脱敏副本
- **访问**：测试团队和产品团队

#### 1.1.3 生产环境 (Production)
- **用途**：正式对外服务
- **配置**：高可用集群配置
- **数据**：真实业务数据
- **访问**：外部用户访问

## 2. 技术栈部署

### 2.1 后端部署 (Spring Boot)

#### 2.1.1 JAR包部署
```bash
# 1. 创建部署目录
sudo mkdir -p /opt/gzang/backend
cd /opt/gzang/backend

# 2. 上传JAR包
scp gzang-server-1.0.0.jar user@server:/opt/gzang/backend/

# 3. 创建配置文件
vim application-prod.yml
```

#### 2.1.2 Docker部署
```dockerfile
# Dockerfile
FROM openjdk:17-jdk-alpine
WORKDIR /app
COPY target/gzang-server-1.0.0.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar", "--spring.profiles.active=prod"]
```

```bash
# 构建镜像
docker build -t gzang/backend:1.0.0 .

# 运行容器
docker run -d \
  --name gzang-backend \
  -p 8080:8080 \
  -v /opt/gzang/config:/app/config \
  -e SPRING_PROFILES_ACTIVE=prod \
  gzang/backend:1.0.0
```

### 2.2 前端部署 (Vue3)

#### 2.2.1 静态资源部署
```bash
# 1. 构建生产版本
npm run build

# 2. 部署到Nginx
sudo cp -r dist/* /usr/share/nginx/html/

# 3. 配置Nginx
server {
    listen 80;
    server_name gzang.com;

    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api/ {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 3. CI/CD流水线

### 3.1 GitHub Actions配置

#### 3.1.1 后端CI/CD
```yaml
# .github/workflows/backend.yml
name: Backend CI/CD

on:
  push:
    branches: [ main, develop ]
    paths: [ 'server/**' ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up JDK 17
        uses: actions/setup-java@v3
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Cache Maven packages
        uses: actions/cache@v3
        with:
          path: ~/.m2
          key: ${{ runner.os }}-m2-${{ hashFiles('**/pom.xml') }}

      - name: Run tests
        run: mvn test

      - name: Build JAR
        run: mvn package -DskipTests

      - name: Build Docker image
        run: |
          docker build -t gzang/backend:${{ github.sha }} ./server
          docker tag gzang/backend:${{ github.sha }} gzang/backend:latest

      - name: Push Docker image
        run: |
          echo ${{ secrets.DOCKER_PASSWORD }} | docker login -u ${{ secrets.DOCKER_USERNAME }} --password-stdin
          docker push gzang/backend:${{ github.sha }}
          docker push gzang/backend:latest
```

#### 3.1.2 前端CI/CD
```yaml
# .github/workflows/frontend.yml
name: Frontend CI/CD

on:
  push:
    branches: [ main, develop ]
    paths: [ 'apps/**', 'packages/**' ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install

      - name: Run tests
        run: pnpm test

      - name: Build
        run: pnpm build

      - name: Deploy to CDN
        run: |
          # 上传到CDN (示例)
          aws s3 sync dist/ s3://gzang-cdn/ --delete
          aws cloudfront create-invalidation --distribution-id ${{ secrets.CDN_DISTRIBUTION_ID }} --paths "/*"
```

### 3.2 部署脚本

#### 3.2.1 蓝绿部署脚本
```bash
#!/bin/bash
# blue-green-deploy.sh

# 配置
APP_NAME="gzang-backend"
BLUE_PORT="8080"
GREEN_PORT="8081"
HEALTH_CHECK_URL="http://localhost"

# 检查当前运行的版本
if docker ps | grep -q "${APP_NAME}-blue"; then
    CURRENT="blue"
    NEXT="green"
    CURRENT_PORT=$BLUE_PORT
    NEXT_PORT=$GREEN_PORT
else
    CURRENT="green"
    NEXT="blue"
    CURRENT_PORT=$GREEN_PORT
    NEXT_PORT=$BLUE_PORT
fi

echo "当前运行版本: $CURRENT (端口: $CURRENT_PORT)"
echo "部署新版本到: $NEXT (端口: $NEXT_PORT)"

# 停止下一个版本 (如果存在)
docker stop ${APP_NAME}-$NEXT 2>/dev/null
docker rm ${APP_NAME}-$NEXT 2>/dev/null

# 启动新版本
docker run -d \
  --name ${APP_NAME}-$NEXT \
  -p $NEXT_PORT:8080 \
  -e SPRING_PROFILES_ACTIVE=prod \
  gzang/backend:latest

# 等待健康检查
echo "等待应用启动..."
for i in {1..30}; do
    if curl -f ${HEALTH_CHECK_URL}:${NEXT_PORT}/actuator/health > /dev/null 2>&1; then
        echo "应用启动成功"
        break
    fi
    sleep 10
done

# 切换流量 (Nginx配置更新)
sed -i "s/proxy_pass http:\/\/localhost:$CURRENT_PORT;/proxy_pass http:\/\/localhost:$NEXT_PORT;/" /etc/nginx/conf.d/gzang.conf
nginx -s reload

# 停止旧版本
docker stop ${APP_NAME}-$CURRENT
docker rm ${APP_NAME}-$CURRENT

echo "部署完成！"
```

## 5. 备份与恢复

### 5.1 数据库备份策略

#### 5.1.1 自动备份脚本
```bash
#!/bin/bash
# mysql_backup.sh

BACKUP_DIR="/opt/backups/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="${BACKUP_DIR}/gzang_${DATE}.sql"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 执行备份
mysqldump -h localhost -u gzang -p${MYSQL_PASSWORD} gzang > $BACKUP_FILE

# 压缩备份文件
gzip $BACKUP_FILE

# 删除7天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

echo "备份完成: ${BACKUP_FILE}.gz"
```

#### 5.1.2 Redis备份配置
```redis.conf
# Redis持久化配置
save 900 1      # 900秒内至少有1个key变化
save 300 10     # 300秒内至少有10个key变化
save 60 10000   # 60秒内至少有10000个key变化

# AOF持久化
appendonly yes
appendfilename "appendonly.aof"
appendfsync everysec
```

## 6. 安全配置

### 6.1 网络安全

#### 6.1.1 Nginx安全配置
```nginx
# nginx.conf
server {
    listen 443 ssl http2;
    server_name gzang.com;

    # SSL配置
    ssl_certificate /etc/ssl/certs/gzang.crt;
    ssl_certificate_key /etc/ssl/private/gzang.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE-RSA-AES256-GCM-SHA384;

    # 安全头
    add_header X-Frame-Options DENY;
    add_header X-Content-Type-Options nosniff;
    add_header X-XSS-Protection "1; mode=block";
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains";

    # 请求限制
    limit_req zone=api burst=10 nodelay;
    limit_req zone=static burst=20 nodelay;

    # CORS配置
    location /api/ {
        if ($request_method = 'OPTIONS') {
            add_header Access-Control-Allow-Origin *;
            add_header Access-Control-Allow-Methods 'GET, POST, PUT, DELETE, OPTIONS';
            add_header Access-Control-Allow-Headers 'Authorization, Content-Type';
            return 204;
        }
    }
}
```

#### 6.1.2 防火墙配置
```bash
# UFW防火墙规则

ufw default allow outgoing

# 允许SSH
ufw allow ssh

# 允许应用端口 (仅内网)

```

### 6.2 应用安全

#### 6.2.1 Spring Security配置
```java
@Configuration
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf().disable()
            .cors().and()
            .sessionManagement().sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            .and()
            .authorizeHttpRequests(authz -> authz
                .requestMatchers("/api/v1/auth/**").permitAll()
                .requestMatchers("/actuator/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            )
            .addFilterBefore(jwtAuthenticationFilter(), UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }
}
```

### 8.2 紧急恢复流程

#### 8.2.1 服务宕机恢复
1. **立即响应**：收到告警后5分钟内响应
2. **问题诊断**：检查应用日志和系统资源
3. **服务重启**：尝试重启应用服务
4. **回滚部署**：如有必要，回滚到上一版本
5. **数据恢复**：从备份恢复数据
6. **问题根因分析**：事后分析并制定预防措施

#### 8.2.2 数据丢失恢复
1. **评估影响**：确定数据丢失范围和影响程度
2. **停止服务**：暂停相关服务写入操作
3. **数据恢复**：从备份恢复数据
4. **数据校验**：验证恢复数据的完整性
5. **服务重启**：恢复正常服务
6. **预防措施**：加强备份策略和监控
