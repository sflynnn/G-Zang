## 1. OWASP Top 10 预防检查

### A01 访问控制失效

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 所有 API 接口都有权限检查 | ☐ | 使用 `@RequirePermission` 注解 |
| 多租户数据隔离正确实现 | ☐ | TenantContextHolder + MyBatis Plus 拦截器 |
| 数据范围过滤覆盖所有查询接口 | ☐ | DataScopeInterceptor |
| CORS 配置仅允许授权域名 | ☐ | 禁止 `allowedOrigins("*")` |
| 管理接口路径不暴露公网 | ☐ | Nginx 配置路径限制或内网访问 |

### A02 加密失败

| 检查项 | 状态 | 说明 |
|--------|------|------|
| HTTPS 强制启用（TLS 1.3） | ☐ | Nginx HSTS 配置 |
| 数据库密码不硬编码 | ☐ | 环境变量或密钥管理服务 |
| 敏感字段（身份证、银行卡）加密存储 | ☐ | AES-GCM 加密 |
| 密码使用 BCrypt 哈希 | ☐ | 禁止 MD5/SHA1 |
| JWT 密钥不硬编码 | ☐ | 从环境变量读取 |

### A03 注入

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 所有 SQL 使用参数化查询 | ☐ | MyBatis Plus Lambda QueryWrapper |
| 无字符串拼接 SQL | ☐ | 代码审查重点检查 |
| 用户输入经过验证和清洗 | ☐ | Hibernate Validator + OWASP Encoder |
| 文件上传验证类型和内容 | ☐ | 白名单 + MIME + 魔数验证 |

### A04 不安全设计

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 已进行威胁建模 | ☐ | 参考 `threat-model.md` |
| 关键流程有安全评审 | ☐ | 认证、支付、数据导出 |
| 无硬编码凭证 | ☐ | Gitleaks 扫描通过 |
| 错误信息不泄露实现细节 | ☐ | 全局异常处理器统一响应 |

### A05 安全配置错误

| 检查项 | 状态 | 说明 |
|--------|------|------|
| Nginx 安全响应头配置 | ☐ | CSP、X-Frame-Options 等 |
| 生产环境关闭调试模式 | ☐ | `debug=false` |
| 默认账户密码已修改 | ☐ | 数据库初始化脚本 |
| 不必要端口未暴露 | ☐ | 防火墙规则 |
| 容器以非 root 用户运行 | ☐ | Dockerfile `USER appuser` |

### A06 脆弱的组件

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 依赖无已知 CVE | ☐ | Trivy/Snyk 扫描通过 |
| 依赖版本定期更新 | ☐ | 季度更新计划 |
| 无使用已废弃组件 | ☐ | Fastjson 1.x 禁止 |
| 组件签名验证 | ☐ | NPM/Maven checksum |

### A07 认证失败

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 登录失败次数限制 | ☐ | 5次后锁定30分钟 |
| Token 有效期 ≤ 2小时 | ☐ | JWT 配置 |
| 会话正确失效 | ☐ | 退出登录清除会话 |
| 多设备登录有安全机制 | ☐ | Token 版本号验证 |
| 密码强度符合策略 | ☐ | 8+字符，含大小写+数字+特殊字符 |

### A08 数据完整性失败

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 关键操作有审计日志 | ☐ | AOP 切面记录 |
| 数据库字段有约束 | ☐ | NOT NULL、UNIQUE CHECK |
| 事务边界正确 | ☐ | ACID 特性 |
| 余额计算使用 DECIMAL | ☐ | 禁止浮点数 |

### A09 日志监控失败

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 安全事件日志记录 | ☐ | 登录、权限拒绝、敏感操作 |
| 日志不记录敏感数据 | ☐ | 密码、Token 脱敏 |
| 有异常检测告警 | ☐ | Prometheus 告警规则 |
| 日志集中存储可查询 | ☐ | ELK Stack |

### A10 服务端请求伪造（SSRF）

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 不允许用户指定服务器地址 | ☐ | Webhook、外部 API 调用 |
| 外部请求 URL 白名单验证 | ☐ | URL.parse + scheme 检查 |
| 禁止访问内网 IP 段 | ☐ | 10.x、172.16.x、192.168.x |

## 2. 安全开发规范

### 代码层面

```java
// ✅ SQL 注入防护
@Select("SELECT * FROM t_user WHERE id = #{id}")
User selectById(@Param("id") Long id);

// ❌ SQL 注入风险
@Select("SELECT * FROM t_user WHERE id = " + id)

// ✅ XSS 防护
import org.owasp.encoder.Encode;
model.addAttribute("input", Encode.forHtmlContent(userInput));

// ✅ 敏感数据脱敏
@JsonSerialize(using = DesensitizeSerializer.class)
private String phone; // 输出: 138****5678
```

### 配置层面

```nginx
# ✅ Nginx 安全响应头
add_header X-Content-Type-Options "nosniff" always;
add_header X-Frame-Options "DENY" always;
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
add_header Content-Security-Policy "default-src 'self'" always;
```
