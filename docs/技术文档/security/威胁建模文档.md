# G-Zang 安全威胁建模

> **文档版本**：1.0.0
> **最后更新**：2026-03-27
> **维护人员**：安全工程师
> **关联规则**：`security.mdc`、`.cursor/rules/security.mdc`

## 1. 系统概述

### 1.1 架构类型

- **架构模式**：微前端 + Spring Boot 微服务
- **数据分类**：PII（个人信息）、财务数据、业务数据
- **信任边界**：用户浏览器 → Nginx 网关 → API 网关 → 后端服务 → 数据库

### 1.2 核心组件

| 组件 | 描述 | 信任级别 |
|------|------|---------|
| Nginx 网关 | 负载均衡、安全响应头、CORS | 边界 |
| Spring Boot API | 业务逻辑、JWT 认证 | 高 |
| MySQL | 数据持久化 | 最高 |
| Redis | 会话、缓存 | 高 |
| 阿里云 OSS | 文件存储 | 中 |

## 2. STRIDE 威胁分析

| 威胁类型 | 目标组件 | 风险等级 | 缓解措施 |
|---------|---------|---------|---------|
| **Spoofing**（欺骗） | 认证接口 `/auth/login` | 高 | BCrypt 密码哈希、失败锁定、JWT Token 绑定 |
| **Tampering**（篡改） | 所有 API 接口 | 高 | 参数化查询、HTTPS、TLS 1.3 |
| **Repudiation**（抵赖） | 所有写接口 | 中 | 不可变审计日志、操作日志记录 |
| **Information Disclosure**（信息泄露） | 用户接口、数据库 | 高 | 数据脱敏、AES 加密、HTTPS |
| **Denial of Service**（拒绝服务） | 公开接口 | 中 | API 限流、WAF、CQRS |
| **Elevation of Privilege**（权限提升） | 管理接口 `/admin/**` | 严重 | RBAC、数据范围过滤、多租户隔离 |

## 3. 攻击面分析

### 3.1 外部攻击面

| 接口 | 攻击向量 | 风险 |
|------|---------|------|
| `/auth/login` | 暴力破解、凭证填充 | 高 |
| `/auth/register` | 账户枚举、垃圾注册 | 中 |
| `/api/**` | SQL 注入、XSS、参数篡改 | 高 |
| 文件上传接口 | webshell 上传、路径穿越 | 严重 |
| API 文档 | 接口探测、信息收集 | 低 |

### 3.2 内部攻击面

| 路径 | 风险 |
|------|------|
| 服务间通信（Feign） | 中间人攻击、Token 窃取 |
| 数据库连接 | SQL 注入、内部数据泄露 |
| Redis 连接 | 未授权访问、缓存投毒 |
| 定时任务 | 权限提升、敏感数据访问 |

## 4. 数据流威胁分析

### 4.1 认证数据流

```
用户 → [浏览器] → HTTPS → [Nginx] → [Spring Boot /auth/login] → [MySQL 用户表]
         ↓                                                          ↓
    JWT Token ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ←
```

**威胁**：
- Token 在传输过程中被截获（缓解：HTTPS 强制）
- Token 被 CSRF 攻击利用（缓解：SameSite Cookie + CSRF Token）
- Token 被 XSS 攻击窃取（缓解：HttpOnly Cookie + CSP）

### 4.2 交易数据流

```
用户 → [浏览器] → HTTPS → [Spring Boot /accounting/transaction] → [MySQL t_transaction]
                         ↓                                              ↓
                   JWT 验证                                   租户隔离
                   RBAC 权限检查                              字段级脱敏
```

**威胁**：
- IDOR（越权访问他人数据）（缓解：数据范围过滤 + 所有权验证）
- 金额篡改（缓解：服务端签名验证）
- 权限提升（缓解：RBAC + 数据范围检查）

## 5. 安全控制措施

### 5.1 身份认证控制

| 控制措施 | 实现方式 | 验证方式 |
|---------|---------|---------|
| 强密码策略 | BCrypt（strength=11） | 密码黑名单校验 |
| 登录失败锁定 | Redis 计数器 | 5次失败锁定30分钟 |
| JWT Token 安全 | RS256 算法，2小时有效期 | Token 版本号验证 |
| MFA 多因素认证 | TOTP / SMS（可选） | 敏感操作二次验证 |

### 5.2 访问控制

| 控制措施 | 实现方式 |
|---------|---------|
| RBAC 角色权限 | `@RequirePermission` 注解 + 拦截器 |
| 多租户数据隔离 | MyBatis Plus TenantLine 拦截器 |
| 数据范围过滤 | DataScopeInterceptor（个人/部门/全部） |
| API 接口限流 | Redis + 令牌桶算法 |

### 5.3 输入安全

| 控制措施 | 实现方式 |
|---------|---------|
| SQL 注入防护 | MyBatis Plus Lambda QueryWrapper（参数化） |
| XSS 防护 | OWASP Encoder + jsoup HTML 净化 |
| 文件上传安全 | 扩展名白名单 + MIME 验证 + 文件头魔数 |
| 参数校验 | `@Valid` + Hibernate Validator |

### 5.4 数据安全

| 控制措施 | 实现方式 |
|---------|---------|
| 传输加密 | HTTPS + TLS 1.3 |
| 敏感数据加密 | AES-GCM（身份证、银行卡） |
| 数据脱敏 | `@JsonSerialize(using=DesensitizeSerializer)` |
| 审计日志 | AOP 切面记录所有写操作 |
