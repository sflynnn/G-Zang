## 1. 环境规划

### 1.1 环境类型

#### 1.1.1 开发环境 (Development)
- **用途**：开发调试、单元测试
- **配置**：最低配置要求，快速启动
- **数据**：测试数据，允许数据丢失
- **访问**：仅开发团队内部访问

#### 1.1.2 测试环境 (Testing)
- **用途**：集成测试、系统测试、验收测试
- **配置**：与生产环境一致
- **数据**：生产数据脱敏
- **访问**：测试团队和产品团队

#### 1.1.3 预发布环境 (Staging)
- **用途**：上线前最终验证
- **配置**：与生产环境完全一致
- **数据**：生产数据脱敏副本
- **访问**：测试团队和技术负责人

#### 1.1.4 生产环境 (Production)
- **用途**：正式对外服务
- **配置**：高可用集群配置
- **数据**：真实业务数据
- **访问**：外部用户访问

## 2. 部署流程

### 2.0 腾讯云服务器初始化

#### 2.0.1 系统初始化配置

```bash
# 1. 使用 SSH 连接服务器（腾讯云控制台获取密码）
ssh root@175.178.9.121

# 2. 创建部署用户（避免使用 root 运行容器）
useradd -m -s /bin/bash deploy
usermod -aG sudo deploy

# 3. 配置 SSH 密钥登录（可选但推荐）
mkdir -p ~/.ssh
chmod 700 ~/.ssh
vim ~/.ssh/authorized_keys  # 粘贴公钥
chmod 600 ~/.ssh/authorized_keys

# 4. 更新系统
apt update && apt upgrade -y

# 5. 设置时区
timedatectl set-timezone Asia/Shanghai

# 6. 配置防火墙（可选，腾讯云安全组已提供网络层防护）
ufw allow 22/tcp    # SSH
ufw allow 80/tcp    # HTTP
ufw allow 443/tcp   # HTTPS
ufw enable
```

#### 2.0.2 安装 Docker 和 Docker Compose

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 启动 Docker 并设置开机自启
systemctl start docker
systemctl enable docker

# 安装 Docker Compose（v2）
apt install docker-compose-plugin -y
docker compose version

# 将 deploy 用户添加到 docker 组
usermod -aG docker deploy
```

#### 2.0.3 腾讯云安全组配置

在腾讯云控制台 → 轻量应用服务器 → 防火墙中添加入站规则：

| 协议 | 端口 | 来源 | 说明 |
|------|------|------|------|
| TCP | 22 | 你的 IP | SSH 管理 |
| TCP | 80 | 0.0.0.0/0 | HTTP Web 访问 |
| TCP | 443 | 0.0.0.0/0 | HTTPS Web 访问 |
| TCP | 8080 | 127.0.0.1 | 后端 API（仅本地访问） |
| TCP | 3306 | 127.0.0.1 | MySQL（仅本地访问） |
| TCP | 6379 | 127.0.0.1 | Redis（仅本地访问） |

```yaml
# /opt/gzang/docker-compose.yml
version: '3.8'

services:
  # MySQL 数据库
  mysql:
    image: mysql:8.0
    container_name: gzang-mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: gzang
      MYSQL_USER: gzang
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
      TZ: Asia/Shanghai
    volumes:
      - ./data/mysql:/var/lib/mysql
      - ./app/server/src/main/resources/db/migration:/flyway/sql:ro
    ports:
      - "127.0.0.1:3306:3306"
    command: --default-authentication-plugin=mysql_native_password
    networks:
      - gzang-network
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  # Redis 缓存
  redis:
    image: redis:7-alpine
    container_name: gzang-redis
    restart: always
    command: redis-server --requirepass ${REDIS_PASSWORD}
    volumes:
      - ./data/redis:/data
    ports:
      - "127.0.0.1:6379:6379"
    networks:
      - gzang-network
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 3s
      retries: 5

  # 后端服务
  server:
    build:
      context: ./app/server
      dockerfile: Dockerfile.prod
    container_name: gzang-server
    restart: always
    environment:
      SPRING_PROFILES_ACTIVE: production
      MYSQL_HOST: mysql
      MYSQL_PORT: 3306
      MYSQL_DATABASE: gzang
      MYSQL_USERNAME: gzang
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
      REDIS_HOST: redis
      REDIS_PORT: 6379
      REDIS_PASSWORD: ${REDIS_PASSWORD}
      JWT_SECRET: ${JWT_SECRET}
    volumes:
      - ./logs/server:/app/logs
    ports:
      - "127.0.0.1:8080:8080"
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy
    networks:
      - gzang-network
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/actuator/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 60s

  # Nginx 反向代理
  nginx:
    image: nginx:alpine
    container_name: gzang-nginx
    restart: always
    volumes:
      - ./app/frontend-pc/dist:/usr/share/nginx/html:ro
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/conf.d:/etc/nginx/conf.d:ro
      - ./logs/nginx:/var/log/nginx
    ports:
      - "80:80"
      - "443:443"
    depends_on:
      - server
    networks:
      - gzang-network

networks:
  gzang-network:
    driver: bridge
```

#### 2.2.3 Nginx 配置

```nginx
# /opt/gzang/nginx/nginx.conf
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;

    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css text/xml text/javascript application/json application/javascript application/xml+rss;

    include /etc/nginx/conf.d/*.conf;
}
```

```nginx
# /opt/gzang/nginx/conf.d/gzang.conf
upstream backend {
    server server:8080;
}

server {
    listen 80;
    server_name _;

    # 后端 API 代理
    location /api/ {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        root /usr/share/nginx/html;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

#### 2.2.6 部署脚本

```bash
#!/bin/bash
# /opt/gzang/deploy.sh

# 加载环境变量

echo "========== 部署完成 =========="
```

```yaml
      - "127.0.0.1:3002:3000"

      - prometheus
    networks:
      - gzang-network

networks:
  gzang-network:
```

### 4.2 监控指标

| 指标 | 阈值 | 告警级别 |
|------|------|---------|
| HTTP 错误率（5xx）| > 5% | 严重 |
| P95 响应时间 | > 2 秒 | 告警 |
| JVM 堆内存使用率 | > 85% | 告警 |
| 数据库连接池使用率 | > 90% | 严重 |
| 磁盘使用率 | > 85% | 告警 |
| CPU 使用率 | > 90% | 告警 |

### 4.3 应急响应

| 事件 | 响应时间 | 处理流程 |
|------|---------|---------|
| 服务不可用（P0）| 15 分钟内响应 | 1. 检查容器状态 → 2. 检查日志 → 3. 重启或回滚 |
| 数据库连接失败 | 15 分钟内响应 | 1. 检查 MySQL 容器 → 2. 检查数据卷 → 3. 从备份恢复 |
| 大规模报错 | 30 分钟内响应 | 1. 应急开启降级 → 2. 定位问题 → 3. 发布修复 |

## 5. 安全加固

### 5.1 服务器安全

```bash
# 1. 修改 SSH 端口（可选）
vim /etc/ssh/sshd_config
# 将 Port 22 改为其他端口（如 2222）

# 2. 禁用 root SSH 登录
vim /etc/ssh/sshd_config
# PermitRootLogin no

# 3. 配置 Fail2ban 防暴力破解
apt install fail2ban -y
systemctl enable fail2ban
systemctl start fail2ban

# 4. 启用自动安全更新
apt install unattended-upgrades -y
dpkg-reconfigure unattended-upgrades
```

### 5.2 网络安全

- 所有生产环境通信使用 HTTPS
- 数据库（3306）、Redis（6379）仅允许本地访问
- 后端服务（8080）仅通过 Nginx 反向代理访问

## 6. 备份与恢复

### 6.1 备份策略

| 数据类型 | 备份频率 | 保留时间 | 存储位置 |
|----------|---------|---------|---------|
| 数据库全量 | 每天 02:00 | 30 天 | /opt/gzang/backup + 本地硬盘 |
| 数据库增量 | 每小时 | 7 天 | /opt/gzang/backup |
| Redis 数据 | 每天 03:00 | 7 天 | /opt/gzang/backup |
| 配置文件 | 每次修改 | 30 天 | /opt/gzang/backup |

### 6.2 自动备份脚本

```bash
#!/bin/bash
# /opt/gzang/backup/backup.sh

set -e

BACKUP_DIR="/opt/gzang/backup"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 加载环境变量
export $(cat /opt/gzang/.env | grep -v '^#' | xargs)

echo "========== 开始备份 =========="

# 1. 数据库备份
echo "备份 MySQL 数据库..."
docker compose exec -T mysql mysqldump -u root -p$MYSQL_ROOT_PASSWORD gzang | gzip > $BACKUP_DIR/mysql_gzang_${DATE}.sql.gz

# 2. Redis 备份
echo "备份 Redis 数据..."
docker compose exec -T redis redis-cli -a $REDIS_PASSWORD BGREWRITEAOF
sleep 5
docker compose exec -T redis redis-cli -a $REDIS_PASSWORD SAVE
docker compose cp redis:/data/dump.rdb $BACKUP_DIR/redis_gzang_${DATE}.rdb

# 3. 配置文件备份
echo "备份配置文件..."
cp /opt/gzang/.env $BACKUP_DIR/env_${DATE}.bak
cp -r /opt/gzang/nginx $BACKUP_DIR/nginx_${DATE}

# 4. 清理旧备份（保留 30 天）
echo "清理超过 30 天的备份..."
find $BACKUP_DIR -name "*.gz" -mtime +30 -delete
find $BACKUP_DIR -name "*.rdb" -mtime +30 -delete
find $BACKUP_DIR -name "*.bak" -mtime +30 -delete

# 5. 列出备份文件
echo "========== 当前备份 =========="
ls -lh $BACKUP_DIR

echo "========== 备份完成 =========="
```

```bash
# 添加定时任务
crontab -e

# 每天凌晨 2 点执行数据库备份
0 2 * * * /opt/gzang/backup/backup.sh >> /opt/gzang/logs/backup.log 2>&1
```

### 6.3 恢复演练

```bash
# 从备份恢复数据库

# 从备份恢复 Redis

```
