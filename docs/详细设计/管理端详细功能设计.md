# G-Zang 管理端平台详细功能设计

> **文档版本**：1.0.0
> **最后更新**：2026-03-27
> **维护人员**：前端开发工程师
> **关联规则**：`admin.mdc`

## 1. 平台概述

### 1.1 定位与目标用户
管理端是G-Zang系统的核心管理平台，主要服务于：
- **超级管理员**：系统级管理员，拥有全部权限
- **公司管理员**：企业级管理员，管理本公司事务

### 1.2 主要功能
- 用户和权限管理系统
- 公司组织架构管理
- 系统配置和监控
- 数据统计和报表

### 1.3 技术架构
- **前端框架**：Vue 3 + TypeScript + Naive UI
- **后端服务**：Spring Boot + MyBatis Plus
- **权限控制**：RBAC角色权限模型
- **数据隔离**：支持多租户架构

## 2. 用户认证与权限系统

### 2.1 登录页面
**页面路径**：`/login`

#### 前端功能
- 用户名/密码登录表单
- 表单验证规则：
  - 用户名：必填，手机号或邮箱格式
  - 密码：必填，6-20位字符
- 记住密码功能
- 登录按钮状态管理（加载/禁用）
- 错误提示显示
- 登录成功后自动跳转到首页

#### 后端API
```javascript
// 登录接口
POST /api/v1/auth/login
Request:
{
  "username": "admin@gzang.com",
  "password": "123456"
}

Response:
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "refresh_token_here",
    "user": {
      "id": 1,
      "username": "admin@gzang.com",
      "nickname": "系统管理员",
      "roleId": 1,
      "companyId": null,
      "permissions": ["USER_MANAGE", "ROLE_MANAGE", "COMPANY_MANAGE"]
    }
  }
}
```

#### 交互流程
1. 用户输入用户名密码
2. 前端表单验证
3. 调用登录API
4. 后端验证用户身份
5. 返回JWT Token
6. 前端存储Token到localStorage
7. 跳转到主页面

## 4. 用户管理模块

### 4.1 用户列表页面
**页面路径**：`/admin/users`

#### 页面布局
- 页面头部：标题 + 添加用户按钮
- 筛选区域：用户名、状态、角色、注册时间范围
- 数据表格：用户列表展示
- 分页组件：页码、每页条数选择

#### 表格列定义
| 列名 | 类型 | 宽度 | 功能 |
|------|------|------|------|
| 选择框 | Checkbox | 60px | 批量操作选择 |
| 用户ID | Number | 80px | 用户唯一标识 |
| 用户名 | Text | 150px | 用户登录名 |
| 昵称 | Text | 120px | 用户显示名称 |
| 角色 | Tag | 100px | 用户角色标签 |
| 所属公司 | Text | 150px | 公司名称（个人用户显示"-"） |
| 状态 | Switch | 80px | 启用/禁用状态 |
| 注册时间 | DateTime | 150px | 用户注册时间 |
| 操作 | Buttons | 200px | 编辑/删除/重置密码 |

#### 前端功能
- **表格功能**：
  - 分页显示（默认20条/页，可选10/20/50/100）
  - 列排序（用户名、注册时间）
  - 表格密度调节（紧凑/默认/宽松）
  - 列显示隐藏控制

- **筛选功能**：
  - 用户名模糊搜索
  - 状态筛选（全部/正常/禁用）
  - 角色筛选下拉框
  - 注册时间范围选择器
  - 公司筛选（管理员可见）

- **批量操作**：
  - 批量启用/禁用
  - 批量删除
  - 批量修改角色

- **单行操作**：
  - 编辑用户信息
  - 删除用户（确认对话框）
  - 重置密码

#### 后端API

**用户列表查询**：
```javascript
GET /api/v1/admin/users?page=1&size=20&username=张三&status=1&roleId=2&startDate=2024-01-01&endDate=2024-12-31

Response:
{
  "code": 0,
  "data": {
    "total": 156,
    "records": [
      {
        "id": 1,
        "username": "zhangsan",
        "nickname": "张三",
        "roleId": 2,
        "roleName": "管理员",
        "companyId": null,
        "companyName": null,
        "status": 1,
        "createTime": "2024-01-15 10:30:00"
      }
    ]
  }
}
```

**用户状态修改**：
```javascript
PUT /api/v1/admin/users/{id}/status
Request: { "status": 0 }

Response: { "code": 0, "message": "操作成功" }
```

**批量操作**：
```javascript
POST /api/v1/admin/users/batch/status
Request: { "userIds": [1,2,3], "status": 0 }

POST /api/v1/admin/users/batch/delete
Request: { "userIds": [1,2,3] }
```

#### 数据权限控制
- **超级管理员**：查看所有用户
- **公司管理员**：只能查看本公司用户

### 4.2 添加用户页面
**页面路径**：`/admin/users/add`

#### 表单字段
| 字段名 | 类型 | 必填 | 验证规则 | 说明 |
|--------|------|------|----------|------|
| 用户名 | Input | 是 | 手机号/邮箱格式，唯一性校验 | 登录用户名 |
| 密码 | Password | 是 | 6-20位字符 | 登录密码 |
| 确认密码 | Password | 是 | 与密码一致 | 密码确认 |
| 昵称 | Input | 否 | 2-20字符 | 显示名称 |
| 角色 | Select | 是 | - | 角色选择 |
| 所属公司 | Select | 否 | 公司管理员必填 | 公司选择 |
| 头像 | Upload | 否 | 图片格式，大小限制 | 用户头像 |
| 状态 | Switch | 否 | 默认启用 | 用户状态 |

#### 前端功能
- 表单实时验证
- 密码强度提示
- 用户名唯一性异步校验
- 角色权限预览
- 表单数据重置
- 提交成功提示和跳转

#### 后端API
```javascript
POST /api/v1/admin/users
Request:
{
  "username": "zhangsan@gzang.com",
  "password": "password123",
  "nickname": "张三",
  "roleId": 2,
  "companyId": 1,
  "avatar": "avatar_url",
  "status": 1
}

Response:
{
  "code": 0,
  "message": "用户创建成功",
  "data": { "userId": 123 }
}
```

### 4.3 编辑用户页面
**页面路径**：`/admin/users/edit/{id}`

#### 功能差异
- 表单数据回填
- 用户名不可编辑
- 密码字段可选（留空表示不修改）
- 更新时间显示
- 操作日志展示

#### 后端API
```javascript
GET /api/v1/admin/users/{id}  // 获取用户详情

PUT /api/v1/admin/users/{id}  // 更新用户信息
```

## 9. 前后端技术实现要点

### 9.1 前端实现要点
- **状态管理**：使用Pinia管理用户权限状态
- **路由守卫**：基于用户权限控制页面访问
- **组件复用**：通用表格、表单、弹窗组件
- **权限指令**：v-permission自定义指令
- **API封装**：统一的请求拦截和错误处理

### 9.2 后端实现要点
- **权限拦截器**：方法级权限校验
- **数据权限**：多租户数据过滤
- **操作日志**：关键操作自动记录
- **缓存策略**：用户信息和权限缓存
- **事务管理**：数据一致性保证

### 9.3 安全考虑
- **XSS防护**：输入数据过滤
- **CSRF防护**：Token验证
- **SQL注入防护**：参数化查询
- **敏感数据脱敏**：日志中敏感信息处理
