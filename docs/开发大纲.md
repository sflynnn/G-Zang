# G-Zang (归藏) 开发大纲

## 1. 项目愿景
**G-Zang (归藏)** 是一款全场景财务管理系统，旨在通过“平台隔离、业务解耦”的架构，为个人提供便捷的日常记账，为修理厂等小微企业提供专业的经营核算。

## 2. 核心技术栈
- **前端**：Vue 3 (Composition API), Vite, Pinia, Vue Router.
- **微前端**：qiankun (仅用于 PC 用户端业务解耦).
- **UI 库**：Naive UI (PC/Admin), Vant (Mobile).
- **后端**：Java 17, Spring Boot 3, MyBatis Plus.
- **数据库**：MySQL 8.0, Redis.
- **规范**：TypeScript, ESLint, Prettier.

## 3. 阶段性开发计划

### 第一阶段：基础设施 (Week 1)
- [ ] 初始化 Monorepo 仓库结构。
- [ ] 搭建 Java 后端基础框架（Security, JWT, MyBatis Plus）。
- [ ] 编写数据库初始化脚本。
- [ ] 封装前端 `shared` 模块（API 请求、通用工具）。

### 第二阶段：用户与权限 (Week 2)
- [ ] 实现统一登录与注册功能。
- [ ] 完成管理端用户管理、角色分配、公司/组织管理。
- [ ] 实现后端多租户（Tenant）数据隔离逻辑。

### 第三阶段：核心记账业务 (Week 3-4)
- [ ] **移动端**：实现个人/公司快速记账、拍照识单。
- [ ] **PC 端**：搭建 qiankun 主应用，集成个人记账与公司业务子应用。
- [ ] **业务逻辑**：修理厂维修支出关联工单、配件成本核算。

### 第四阶段：报表与优化 (Week 5)
- [ ] 集成 ECharts，实现多维度财务分析报表。
- [ ] 实现数据导出（Excel/PDF）。
- [ ] 全平台联调与性能优化。

## 4. 关键目录结构
```text
G-Zang/
├── apps/
│   ├── admin/          # 管理端 (Vue3)
│   ├── pc-main/        # PC端主应用 (Vue3 + qiankun)
│   ├── pc-personal/    # 个人记账子应用 (Vue3)
│   ├── pc-business/    # 公司业务子应用 (Vue3)
│   └── mobile/         # 移动端 (Vue3 + Vant)
├── packages/
│   └── shared/         # 跨端共享模块 (TS, API, Utils)
├── server/             # Java 后端 (Spring Boot)
└── docs/               # 项目文档
```
