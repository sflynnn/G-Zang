> **最后更新**：2026-03-27
> **维护人员**：开发团队
> **关联规则**：`backend.mdc`、`mdc_rules.mdc`

## 1. 前端命名规范

### 1.1 文件命名

| 类型 | 命名格式 | 示例 |
|------|---------|------|
| Vue 组件文件 | PascalCase | `UserList.vue`、`AccountCard.vue` |
| 页面组件 | kebab-case | `user-list.vue`、`account-detail.vue` |
| 工具函数文件 | camelCase + 后缀 | `dateUtils.ts`、`formatters.ts` |
| 类型定义文件 | PascalCase + `.type.ts` | `User.type.ts` |
| 组合式函数文件 | camelCase + `.ts` | `useVoiceRecord.ts` |
| 测试文件 | 同名 + `.spec.ts` | `useVoiceRecord.spec.ts` |
| Store 文件 | camelCase + `.ts` | `userStore.ts` |

### 1.2 变量命名

| 类型 | 命名格式 | 示例 |
|------|---------|------|
| 常量 | UPPER_SNAKE_CASE | `API_BASE_URL`、`MAX_RETRY_COUNT` |
| 变量/函数 | camelCase | `userList`、`getUserById()` |
| 组件引用 | camelCase | `userList`、`accountCard` |
| 类型/接口 | PascalCase | `User`、`ApiResponse<T>` |
| 枚举 | PascalCase | `TransactionType`、`UserStatus` |
| 事件处理函数 | `handle` + 动作名 | `handleSubmit`、`handleClick` |
| 异步获取函数 | `fetch`/`load` + 名词 | `fetchUserInfo`、`loadTransactions` |

### 1.3 CSS 类命名

使用 Tailwind CSS 原子类。对于自定义类：

```css
/* 组件样式 */
.gzang-card { }
.gzang-btn-primary { }
.gzang-input { }
.gzang-table { }

/* 状态修饰 */
.gzang-text-success { color: var(--gzang-success); }
.gzang-text-danger { color: var(--gzang-danger); }
.gzang-bg-surface { background-color: var(--gzang-surface); }
```

## 2. 后端命名规范

### 2.1 Java 类命名

| 类型 | 命名格式 | 示例 |
|------|---------|------|
| 控制器类 | `XxxController` | `UserController` |
| 服务接口 | `XxxService` | `UserService` |
| 服务实现 | `XxxServiceImpl` | `UserServiceImpl` |
| 实体类 | `Xxx` | `User`、`Transaction` |
| DTO 类 | `XxxDTO` | `CreateUserDTO`、`LoginDTO` |
| VO 类 | `XxxVO` | `UserVO`、`TransactionVO` |
| 异常类 | `XxxException` | `BusinessException`、`AuthException` |
| 工具类 | `XxxUtil` | `JwtUtil`、`DateUtil` |
| 常量类 | `XxxConstants` | `ErrorCode`、`PermissionCode` |

### 2.2 Java 方法命名

| 操作类型 | 命名格式 | 示例 |
|---------|---------|------|
| 查询方法 | `getXxx()` / `findXxx()` / `selectXxx()` | `getUserById()` |
| 新增方法 | `createXxx()` / `saveXxx()` | `createTransaction()` |
| 更新方法 | `updateXxx()` / `modifyXxx()` | `updateUserInfo()` |
| 删除方法 | `deleteXxx()` / `removeXxx()` | `deleteTransaction()` |
| 业务方法 | 动词 + 名词 | `approveOrder()`、`syncOfflineData()` |
| 布尔判断 | `isXxx()` / `hasXxx()` / `checkXxx()` | `isAdmin()`、`hasPermission()` |

## 3. 数据库命名规范

### 3.1 表命名

```
前缀: t_
命名: 小写字母 + 下划线
格式: t_业务模块_实体名

t_user                    # 用户表
t_transaction             # 交易表
t_role_permission          # 角色权限关联表
t_company                 # 公司表
```

### 3.2 字段命名

```
命名: 小写字母 + 下划线

主键: id (BIGINT, 自增)
外键: xxx_id (e.g., user_id, company_id)
时间: create_time, update_time, delete_time
状态: status (TINYINT)
逻辑删除: deleted (TINYINT, 0=存在, 1=删除)
金额: amount (DECIMAL(19,4))
```

### 3.3 索引命名

```
主键索引: PRIMARY
唯一索引: uk_{表名}_{字段名}
普通索引: idx_{表名}_{字段名}
复合索引: idx_{表名}_{字段1}_{字段2}
```
