# G-Zang 数据库初始化指南

## 📋 概述

本指南介绍如何在数据库中初始化G-Zang系统的基本数据，包括角色、权限、用户等。

## 🚀 执行步骤

### 1. 数据库准备
确保MySQL数据库已创建，并配置正确的字符集：
```sql
CREATE DATABASE IF NOT EXISTS g_zang CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2. 创建表结构
首先执行建表SQL创建所有数据库表：
```bash
mysql -u username -p g_zang < docs/create-tables.sql
```

或者在MySQL命令行中：
```sql
USE g_zang;
source docs/create-tables.sql;
```

### 3. 创建索引（可选，但推荐）
执行索引创建脚本，为表添加必要的索引以提高性能：
```bash
mysql -u username -p g_zang < docs/create-indexes.sql
```

或者在MySQL命令行中：
```sql
USE g_zang;
source docs/create-indexes.sql;
```

**注意**：索引脚本使用了 `IF NOT EXISTS` 语法，可以安全重复执行。

### 4. 执行初始化数据
表结构和索引创建完成后，执行数据初始化SQL：
```bash
mysql -u username -p g_zang < docs/init-data.sql
```

或者在MySQL命令行中：
```sql
USE g_zang;
source docs/init-data.sql;
```

### 4. 验证结果
执行完成后验证数据是否正确插入（详见下文验证查询）。

## 🔍 验证结果

执行完成后，可以通过以下查询验证数据插入是否成功：

```sql
-- 查看各表数据量
SELECT '角色数量:' as info, COUNT(*) as count FROM t_role
UNION ALL
SELECT '权限数量:', COUNT(*) FROM t_permission
UNION ALL
SELECT '用户数量:', COUNT(*) FROM t_user;

-- 查看角色权限分配
SELECT r.role_name, COUNT(rp.permission_id) as permission_count
FROM t_role r
LEFT JOIN t_role_permission rp ON r.id = rp.role_id
GROUP BY r.id, r.role_name;

-- 查看用户角色分配
SELECT u.username, u.nickname, r.role_name
FROM t_user u
JOIN t_role r ON u.role_id = r.id
ORDER BY u.id;
```

## 🆘 故障排除

### 表不存在错误
如果出现 `Table 'xxx' doesn't exist` 错误：
1. **确认数据库已创建**：`CREATE DATABASE g_zang CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;`
2. **先执行建表SQL**：运行 `docs/create-tables.sql` 创建所有表结构
3. **再执行数据SQL**：运行 `docs/init-data.sql` 插入初始化数据

### 外键约束错误
确保按照正确的顺序执行INSERT语句：角色 → 权限 → 角色权限关联 → 公司 → 用户

### 字符集问题
确保数据库和表的字符集设置为 `utf8mb4`：
```sql
-- 检查数据库字符集
SHOW CREATE DATABASE g_zang;

-- 检查表字符集
SHOW CREATE TABLE t_user;
```

### 密码验证失败
确认使用的BCrypt哈希与后端配置的加密器一致：
```java
// 后端验证代码
BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();
boolean matches = encoder.matches("123456", hashedPassword);
```

### 索引重复错误
如果出现 `Duplicate key name 'xxx'` 错误：
1. **使用单独的索引脚本**：运行 `docs/create-indexes.sql`
2. **手动删除重复索引**：
   ```sql
   DROP INDEX IF EXISTS index_name ON table_name;
   ```
3. **检查现有索引**：
   ```sql
   SHOW INDEX FROM table_name;
   ```

### 权限相关错误
- 确保角色和权限数据已正确插入
- 检查角色权限关联表是否有正确的数据
- 验证用户角色ID是否正确指向t_role表
